Descritivo dos Casos de Uso — Fluxos e Cenários
UC-01 — Cadastrar Conta  (RF01, RF02)
Ator(es)Estabelecimento, Profissional
Pré-condiçõesUsuário não possui cadastro. Acesso à internet disponível.
Fluxo Principal1. Usuário acessa /cadastro e seleciona tipo de conta.
2. Estabelecimento preenche CNPJ, nome fantasia, ramo e endereço.
   Profissional preenche CPF, nome, especialidades e portfólio.
3. Sistema «include» Validar CPF/CNPJ via Receita Federal.
4. Sistema cria conta, gera token de sessão e envia e-mail de confirmação.
5. Usuário é redirecionado ao Dashboard.
Pós-condiçõesConta criada com sucesso. Acesso à plataforma liberado.
Fluxo de Exceção3a. CPF/CNPJ inválido → sistema exibe mensagem de erro e solicita correção.
3b. E-mail já cadastrado → sistema sugere recuperação de senha.
UC-02 — Publicar Vaga  (RF03)
Ator(es)Estabelecimento
Pré-condiçõesEstabelecimento autenticado. Wallet com saldo suficiente ou pré-aprovação.
Fluxo Principal1. Estabelecimento acessa 'Nova Vaga' no dashboard.
2. Preenche: função, data, horário, valor (R$) e requisitos do turno.
3. Confirma publicação.
4. Sistema registra vaga com status = 'Publicado'.
5. Sistema «include» Notificar Profissionais em Tempo Real (RF11) via push notification para profissionais compatíveis.
Pós-condiçõesVaga publicada. Profissionais compatíveis notificados. Status = Publicado.
Fluxo de Exceção4a. Informações incompletas → sistema destaca campos obrigatórios e bloqueia publicação.
UC-03 — Candidatura One-Tap  (RF05)
Ator(es)Profissional
Pré-condiçõesProfissional autenticado. Perfil completo (CPF validado, habilidades cadastradas).
Fluxo Principal1. Profissional recebe notificação de vaga compatível ou navega no feed.
2. Visualiza detalhes da vaga (valor, horário, local).
3. Toca em 'Candidatar' («extend» sobre UC-03: Buscar e Visualizar Vagas).
4. Sistema registra candidatura e notifica Estabelecimento.
5. Status do profissional na vaga → 'Notificado'.
Pós-condiçõesCandidatura registrada. Estabelecimento notificado para aprovar ou recusar.
Fluxo de Exceção4a. Profissional já candidatado para o mesmo horário → sistema bloqueia e exibe conflito.
4b. Perfil incompleto → sistema redireciona para completar perfil.
UC-04 — Check-in / Check-out QR Code  (RF07)
Ator(es)Profissional, Estabelecimento
Pré-condiçõesTurno com status 'Aprovado'. Profissional e Estabelecimento autenticados. GPS habilitado.
Fluxo Principal1. No horário combinado, Profissional abre o app e acessa QR Code de Check-in.
2. Estabelecimento escaneia QR Code do profissional (ou vice-versa).
3. Sistema valida geolocalização e registra checkIn = agora. Status → 'Em Andamento'.
4. Sistema envia confirmação (9a: checkInConfirmado) ao Estabelecimento.
--- execução do turno ---
5. Ao fim do turno, Profissional exibe QR Code de Check-out.
6. Estabelecimento escaneia QR Code de saída.
7. Sistema registra checkOut e «include» Processar Pagamento Automático.
8. Status → 'Aguardando Avaliação'.
Pós-condiçõesTurno registrado. Pagamento split (takeRate 15%) processado. Status = Aguardando Avaliação.
Fluxo de Exceção3a. GPS incompatível com o endereço da vaga → sistema alerta e registra inconsistência para Admin.
6a. QR Code expirado → Estabelecimento solicita novo código via suporte.
UC-05 — Avaliação Bilateral  (RF08, RF09)
Ator(es)Estabelecimento (RF08), Profissional (RF09)
Pré-condiçõesTurno com status 'Aguardando Avaliação'. Período de avaliação ativo (até 24h após check-out).
Fluxo Principal1. Sistema envia solicitação de avaliação para ambos os atores (msg 14 / 14a).
2. Estabelecimento avalia Profissional: nota (1-5) + comentário.
3. Profissional avalia Estabelecimento: nota (1-5) + comentário.
4. Sistema «include» Pagamento Automático: split confirmado, valorLiquido creditado na wallet do Profissional.
5. Sistema atualiza mediaAvaliacao e noShowRate. Status → 'Concluído'.
Pós-condiçõesAvaliações registradas. Wallet do Profissional creditada com valorLiquido = valorBruto × (1 - takeRate).
Fluxo de Exceção3a. Prazo de avaliação expirado → sistema libera pagamento automaticamente sem avaliação.
3b. Conflito reportado → «extend» sobre UC-05 aciona UC-12: Mediação de Conflitos.
UC-06 — Mediação de Conflitos / Moderar  (RF12)
Ator(es)Admin ServLink
Pré-condiçõesAdmin autenticado com nível de acesso moderador. Conflito ou denúncia registrada.
Fluxo Principal1. Admin acessa painel de moderação.
2. Visualiza lista de conflitos/denúncias por prioridade.
3. Seleciona ocorrência e acessa histórico completo do turno.
4. Admin decide: mediação, penalidade, bloqueio ou encerramento.
5. Sistema aplica decisão: atualiza status, notifica partes e registra log de moderação.
Pós-condiçõesConflito resolvido. Partes notificadas. Registro de moderação salvo no histórico.
Fluxo de Exceção4a. Admin não consegue resolver → escalada para equipe jurídica (fluxo externo ao sistema).
